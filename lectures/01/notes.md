**SPEAKING NOTES:**
- Python has become the dominant language in health data science due to its readability and extensive libraries
- Git enables reproducible research by tracking changes and facilitating collaboration
- Markdown provides a simple way to document research and share findings
- The command line offers powerful tools for efficient data processing

**KEY POINTS:**

- Command Line: Efficient data processing

### 3. Command Line Basics (10 minutes)

**SPEAKING NOTES:**
- The command line is essential for working with large health datasets efficiently
- Common commands like `grep` can search through clinical notes for specific terms
- Pipes and redirection allow us to filter and transform health data without loading it into memory
- These skills are particularly valuable when working with sensitive health data that can't be processed in cloud environments
- While the terminal might seem intimidating, it's a powerful tool that becomes second nature with practice

**KEY POINTS:**
- Navigation: `pwd`, `ls`, `cd`
- File operations: `mkdir`, `cp`, `mv`
- Data processing: `grep`, `cut`, `sort`
- Safety: No undo in terminal, be careful

**KEY POINTS:**

### 5. Git & GitHub (15 minutes)

**SPEAKING NOTES:**
- Git enables reproducible research by tracking every change to code and data
- GitHub provides a platform for sharing research while maintaining version control
- The basic workflow of add, commit, and push mirrors the scientific method
- Never commit sensitive health data - Git history is permanent
- GitHub's collaboration features support team-based health research

**KEY POINTS:**
- Track changes over time
- Collaborate with others
- Basic workflow
- Never commit sensitive data
- GitHub for sharing work
